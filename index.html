Sigma16

Sigma16 is a computer architecture designed for research and teaching in computer systems. This application provides a complete environment for experimenting with the architecture, including an editor, assembler, linker, emulator, and an integrated development environment. There is a digital circuit that implements the architecture, which is specified using the Hydra functional hardware description language. Hydra can simulate the circuit, and machine language programs can run on both the emulator and the circuit.

Quick start: launch Sigma16.exe and visit http://localhost:8023 in your browser.

Running the application

You need to start two programs: the Sigma16 application, and a web browser. The application runs as an independent process, and it uses the browser to display the graphical user interface. The software runs on Windows, Macintosh, and Linux, and currently it works with Chrome and Firefox. Unzip the installation file and enter the directory. To run the application you need to do two things:

  1. Launch the program, using any one of the following methods:
    • Run the executable file: click Sigma16.exe
    • If there is no executable file, run the program from source: enter bash run in a shell. (If you're using cygwin, then enter bash runcygwin.)
    • You can compile the program.
    • The program requires several files to be present, and it needs to be able to find them. If it fails to find the required files, the symptom is that the GUI will repeatedly start, fail, and restart.
    • See Installing the Software below, and the User Guide, for more details.
  2. Open the GUI in a browser.
    • Visit the following address in your browser, either Chrome or Firefox: http://localhost:8023
    • If you get an error on your browser, something like This site can't be reached, localhost refused to connect, that means the program isn't running. Launch it and the web page should connect.
    • The application assumes a fairly wide browser window, and it displays the User Guide to the right of the architecture display. If the browser window isn't wide enough, the documentation will not be visible.

Documentation

  • The User Guide is displayed in the running application
  • Without launching the program, you can read the User Guide as html or alternatively as plain text

Using the software

  • Refreshing. The processor pane has a Refresh button which redraws the display. If you refresh the browser window (a little circle with an arrow beside the URL entry), this will restart the application, which is probably not what you want.
  • Finding your files. The best place to keep your assembly language programs is in the Documents folder. Create a folder in your Documents directory called Sigma16programs, and save your programs there. Then when you launch Sigma16, go to the editor pane, click Open, and in the Open dialogue click Documents. It should show a list of directories including Sigma16programs; click that and then the exercise file, then click Open Selected File. Let me know if it works, or if you're still having problems.

Installing the software

The Sigma16 directory may be either a source distribution or a compiled distribution (sometimes called a "binary").

  • A compiled distribution contains an executable program. On Windows this is Sigma16.exe. You can run the application by launching that executable file. For this to work, the executable needs to be the right kind for your operating system.
  • If you don't have an executable program, or if it doesn't match your operating system, you can run the application from the source.
  • There are two ways to run Sigma16 from a source distribution: you can compile it (using ghc), or you can run it from the source program (using ghci). Either way, you need to have the Haskell Platform installed, and it needs to be the May 2016 release (containing compiler version ghc-8.0.1). This is available from https://www.haskell.org/platform/prior.html
  • The program can be compiled using the following commands: 
              ghc --version      (should be 8.0.1)
          cabal update
          cabal configure
          cabal build
          cabal install
          Sigma16
  • To run from source: open a shell and enter the command bash run (if you're using cygwin on Windows, enter bash runcygwin). It will run more slowly than a compiled version but is likely to be fast enough.
